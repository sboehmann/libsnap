size_t snap_lzma_bound ( size_t  size  ) 

Calculate output buffer size for lzma.

When trying to compress uncompressible data, the encoded size will be slightly bigger than the input data. This function calculates how much output buffer space is required for snap_XXX_encode().

The calculated value is not exact, but it is guaranteed to be big enough. The actual maximum output space required may be slightly smaller. This should not be a problem in practice.

Note:
The limit calculated by this function applies only to single-call encoding. Multi-call encoding may (and probably will) have larger maximum expansion when encoding uncompressible data
Parameters:
size Size of the input data
Returns:
Maximum size of output data
